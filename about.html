InfoScope

A Mini News Search Engine built on classic IR principles

(HTML • CSS • JavaScript • TF-IDF • BM25)

This website uses fictional, AI-generated news articles. Do not use them for real-world research.

Overview

InfoScope is a lightweight search engine built for my CIST433: Information Storage & Retrieval final project. It demonstrates how search engines actually think — using techniques like tokenization, stopword removal, and term weighting — all applied to a curated dataset of 400 fictional news article summaries.

The goal isn’t to compete with Google — it’s to show that I understand how Google works.

Information Retrieval Features

  • Tokenization: Splits text into searchable words
  • Stopword Removal: Removes filler words like “the,” “is,” “of”
  • Stemming: Normalizes word forms (running → run)
  • TF-IDF Model: Classic keyword importance weighting
  • BM25-lite: Modern ranking model balancing frequency & length
  • Field Boosts: Prioritizes title/tag matches
  • Boolean Logic: Supports AND, OR, and quoted phrases
  • Snippets & Highlights: Shows keyword context in results

How the Ranking Models Work

TF-IDF

TF-IDF stands for Term Frequency–Inverse Document Frequency. It boosts articles that:

  • use your keyword more often (TF)
  • use keywords that are relatively rare across the dataset (IDF)

In plain English: TF-IDF ranks articles higher when they talk a lot about your query using meaningful, specific terms.

BM25-lite

BM25 is a more modern and realistic ranking formula. It improves TF-IDF by:

  • penalizing overly long articles
  • making sure repeating a word 20 times doesn’t artificially inflate relevance
  • ranking results in a way that tends to match human expectations

In plain English: BM25 feels more “Google-like” — it ranks based on keyword importance, balance, and context.
